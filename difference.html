define Before the advent of ES6, var declarations ruled. let is now preferred for variable declaration Variables declared with the const maintain constant values
Keywords defination `var` is now the weakest signal available when you define a variable in JavaScript. `let` is a signal that the variable may be reassigned, such as a counter in a loop, or a value swap in an algorithm. It also signals that the variable will be used only in the block it’s defined in, which is not always the entire containing function. `const` is a signal that the identifier won’t be reassigned.
Define variables using the keywords var, let or const. var a=10; let b=20; const PI=3.14;
Program Examples Input: console.log(x); var x=5; console.log(x); Output: undefined 5 Input: console.log(x); let x=5; console.log(x); Output: Error const PI = 3.141592653589793; PI = 3.14; PI = PI + 10; output: TypeError: Assignment to constant variable.
